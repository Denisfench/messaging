Review of Telegram Interoperability Characteristics

This document answers some basic questions about Telegram for creating custom GET/POST APIs

  • we can create custom GET/POST APIs in telegram that users can interact with.
  • Telegram allows developers to create bots. Bots are third-party applications that run inside Telegram.
  • Users can interact with bots by sending them messages, commands and inline requests.
  • Telegram Bot API provides a HTTP-based interface for developers keen on building bots for Telegram.
  • A detailed documentation on how to create a bot for telegram can be found here (https://core.telegram.org/bots)

1. How can we achieve calling (custom GET/POST)APIs?

  • Messages, commands and requests sent by users are passed to the software running on our servers.
  • Telegram's intermediary server handles all encryption and communication with the Telegram API for us.
  • we communicate with this server via a simple HTTPS-interface that offers a simplified version of the Telegram API.
  • that interface is called Bot API. (https://core.telegram.org/bots/api)
  • Bot API supports GET and POST HTTP methods. there are four ways of passing parameters in Bot API requests:
  • 1) URL query string
  • 2) application/x-www-form-urlencoded
  • 3) application/json (except for uploading files)
  • 4) multipart/form-data (use to upload files)
  • We can also design a custom app using TDLib (Telegram Database Library), a cross-platform, fully functional Telegram client.
  • It is designed to help third-party developers create their own custom apps using the Telegram platform.
  • creating a custom client is not necessary but can be considered if we want to provide an alternate way to communicate with our bot.

2. What resources are required for producing or providing it?

  • Our bot has to continously pull updates from the Telegram server using one of the methods described below.
  • we will receive JSON-serialized Update objects as a result.
  • There are two mutually exclusive ways of receiving updates for our bot:
  • 1) the getUpdates method (https://core.telegram.org/bots/api#getupdates)
  • 2) Webhooks (https://core.telegram.org/bots/api#setwebhook)
  • Incoming updates are stored on the telegram server until the bot receives them either way.
  • The messages are stored on telegram server for 24 hours max.

3. Are there any standards surrounding the product that we have to follow?

  • At a higher level we need to take care of user authorization and message encription.
  • Telegram follows MTProto Mobile Protocol a detailed description can be found here: https://core.telegram.org/mtproto
  • If we decide to develop a client application we need to comply with the security guidelines found here: https://core.telegram.org/mtproto/security_guidelines
